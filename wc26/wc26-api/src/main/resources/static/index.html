shared-front

shared-front for web

A cross-project frontend package for shared copy, layout decisions, and lightweight helpers.

Shared design language

Keep project-specific UI in one place and re-use the same words, colors, and patterns across apps.

Web and mobile ready

The workspace is split so Next.js and Expo can evolve independently without coupling their build tools.

Workspace-friendly imports

Local packages resolve through pnpm workspaces, so feature code stays close to the project that owns it.
